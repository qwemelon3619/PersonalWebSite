Entry
Browser -> Nginx

Nginx fronts the stack and splits traffic between the SSR app, API Gateway, and image path.

Core
Web / Gateway / Services

`web-front`, `api-gateway`, `auth-service`, `post-service`, and `img-service` each own a clear responsibility.

State
PostgreSQL / Redis / Blob

Persistent data, token lifecycle state, and media storage are externalized from the app processes.

Auth and Token Flow

  1. The login flow starts from `web-front` and redirects through Google OAuth.
  2. `auth-service` validates the callback and issues access and refresh tokens.
  3. Protected write requests pass through `api-gateway` first.
  4. If an access token is expired, the gateway attempts refresh through `auth-service`.
  5. `post-service` receives authenticated write traffic only after gateway validation succeeds.

Post and Image Flow

  1. Authors submit markdown content and optional thumbnail data from the web UI.
  2. `post-service` processes embedded image references before persistence.
  3. `img-service` handles upload and delete responsibilities for binary assets.
  4. Post content stores URLs instead of raw image payloads.
  5. Blob storage uses Azurite in local development and Azure Blob-compatible flow in deployment.

Translation Pipeline

  1. Korean source content is stored as the canonical write result.
  2. The post create or update path returns without waiting on external translation latency.
  3. A background goroutine translates title and HTML-rendered content through DeepL.
  4. Translated English fields are persisted later to support bilingual presentation.

Deployment Topology

  1. Docker Compose provides a full local stack with PostgreSQL, Redis, Azurite, Nginx, and all Go services.
  2. Service boundaries remain explicit in both local and deployment environments.
  3. GitHub Actions runs module tests before image build and push.
  4. The project has been moved toward Oracle Cloud free tier hosting while keeping the service layout intact.

Why This Structure Matters

01
Single auth boundary

Gateway-centered enforcement keeps protected write behavior consistent.

02
Short write path

Async translation prevents external APIs from slowing author actions.

03
Isolated media handling

Image storage is decoupled from post domain logic and database concerns.

04
Operational realism

Compose, CI, externalized state, and health checks make the project portfolio-grade.
